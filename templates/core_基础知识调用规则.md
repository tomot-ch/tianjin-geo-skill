# 基础知识调用规则

## 一、什么时候调用基础知识

遇到以下情况时，调用 `references/core_高中地理核心知识体系.md`：

1. 题目材料较新，真题库中没有完全对应案例；
2. 用户问的是知识点本身；
3. 题目属于基础地理原理，如太阳、地质、人口、城市、区位、碳排放；
4. 现有 chains/ 中没有足够完整的链条；
5. 需要把学生口语化回答改写成规范地理表达。

## 二、调用顺序

```text
题目设问 → 材料条件 → 母题判断 → 真题链条 → 基础知识补充 → 天津语言输出
```

注意：

- 真题链条优先；
- 基础知识补漏洞；
- 答案必须回到材料；
- 不能把基础知识整段照搬。

## 三、基础知识转答案流程

### 1. 找条件

从题目中提取：

- 位置；
- 地形；
- 气候；
- 水文；
- 土壤；
- 植被；
- 资源；
- 交通；
- 市场；
- 政策；
- 科技；
- 人类活动。

### 2. 转含义

把材料语言转成地理术语。

示例：

```text
雨下得急 → 降水集中且多暴雨
```

```text
山坡陡 → 坡度大，径流速度快
```

```text
人多、消费强 → 市场需求大
```

### 3. 串链条

使用：

```text
条件 → 过程 → 结果
```

### 4. 写答案

每一点写成：

```text
材料条件 + 地理过程 + 结果
```

示例：

```text
该地坡度较大，地表径流速度快，流水冲刷能力强，易造成水土流失。
```

## 四、不能这样用

错误示范：

```text
森林有很多生态功能。
```

问题：太空泛，没有回扣材料。

正确示范：

```text
该地森林覆盖率提高后，可截留降水、增加下渗，增强涵养水源和保持水土能力。
```

## 五、输出要求

若用户要求标准答案：

```text
①……
②……
③……
```

若用户要求讲给学生：

```text
这题本质上问的是……
先看材料……
所以答案可以写成……
```

若用户要求训练入库：

```text
母题：
材料条件：
核心链条：
标准话术：
易错点：
可迁移规则：
```
