# 天津高考地理推理系统

本项目不是普通地理笔记，也不是把教材知识点重新抄一遍。

本项目的目标是把历年天津高考地理真题、解析、课堂笔记、学生答案和答题框架，持续蒸馏成一个可以反复迭代的“天津高考地理推理系统”。它关注的不是“记住多少知识点”，而是：

- 看到题目材料后，能识别条件。
- 根据条件推出地理含义。
- 把地理含义串成因果链。
- 用天津卷常见的规范语言生成答案。
- 反过来诊断学生答案为什么丢分。

## 输入

本仓库可以接收以下材料：

- 天津高考地理真题
- 天津各区模拟题、联考试题
- 官方或机构解析
- 课堂笔记、专题讲义
- 学生作答、错题记录

## 输出

每次处理材料后，尽量沉淀为以下几类成果：

- 题目分析：题目类型、材料条件、设问方向、核心推理链。
- 母题归类：把题目归入河流、气候、工业、农业、城市、港口、地貌、生态等专题。
- 因果链：把“条件 → 地理过程 → 结果”整理成可复用链条。
- 天津风格答案：使用高考阅卷更容易接受的规范表达。
- 讲题稿：把复杂推理讲成高中生能听懂的话。
- 学生易错点：记录常见误判、漏答、表达不规范和因果断裂。

## 目录说明

`papers/` 用于存放原始试卷、解析、模拟题、年份材料和上游框架材料。每个年份单独放置，`simulations/` 存放模拟题。

`extracted/` 用于存放从题目中提炼出的母题库。每个专题一个文件，例如河流、工业、气候、农业、城市等。

`chains/` 用于存放因果链库。这里整理的是做题时最关键的“条件如何推出答案”的推理路径。

`templates/` 用于存放答案语言、题目分析、讲题稿、评分诊断等模板。

`skills/` 用于存放可复用的做题能力模块，例如条件识别、材料提取、题型分类、选择题陷阱、综合题求解和教学风格。

`examples/` 用于存放完整案例。每个案例都应展示从材料识别到答案生成的完整工作流。

`references/` 用于存放跨专题参考材料，例如天津卷语言风格、知识框架、答案模板等。

`prompts/` 用于存放可复用 prompt，例如单题分析、材料蒸馏、学生答案诊断和仓库迭代。

## 本轮结构补强

本轮重点补齐以下薄弱处：

- `references/knowledge_framework.md`：把天津卷能力框架整理成“题型入口—材料条件—推理链—答案话术”。
- `references/answer_templates.md`：把原因、影响、措施、意义、区位、图表、分主体等模板做成总索引。
- `references/tianjin_answer_style.md`：沉淀天津卷高频句式、表达规范和失分表达。
- `extracted/`：补齐农业、城市、气候、综合题、地貌、工业、港口交通等母题归纳。
- `chains/`：补齐农业、城市、气候、生态、地貌、工业、港口交通等基础链条。
- `prompts/`：新增可复用的分析、蒸馏、诊断和迭代 prompt。

## 后续迭代原则

1. 一次只处理一个专题，例如只处理河流题，或只处理工业区位题。
2. 不要全仓库反复扫描，优先打开当前专题相关文件，节省 token。
3. 每处理一道题，都优先判断它能补充到哪个母题、哪条因果链、哪类答案语言。
4. 新增内容要能直接服务做题，避免空泛总结。
5. 保留来源信息。后续处理真实试卷时，应记录年份、题号、材料关键词和答案依据。
6. 先小样板，再扩展。每个专题先形成一个能跑通的样板，再逐步加题、加规则、加易错点。
7. 真题原文、解析、蒸馏结果、可复用能力要分层存放，不混写。
8. 涉及学生答案时只保留可匿名化的失分类型和改写规则，不保存隐私信息。
