# 真实讲题案例入库规则

本目录用于存放后续从真实聊天记录、语音转文字或课堂讲题中匿名化整理出的案例。案例文件只保存教学价值，不保存隐私原文。

每个案例文件必须包含：

## 题目编号

如：题目1、题目2、题目3、题目4。

## 题目主题

概括题目核心内容，例如“巴伦支海海洋热量收支与海冰变化”。

## 原始讲解来源

聊天记录 / 语音转文字 / 截图整理。

不要保存未经处理的原始聊天全文。

## 隐私处理

删除学生姓名、头像、学校、手机号、微信号、账号 ID、群名、住址等信息。

建议统一写：

```text
已删除学生姓名、头像、学校、手机号、微信号等信息。
```

## 讲题风格线

提炼老师这道题体现出的讲题动作，例如：

- 先翻译材料。
- 先找图表中最确定项。
- 先承认学生选项有相关性，再指出不是主因。
- 把专业概念翻译成人话。
- 把口语解释转成考试表达。

## 知识点线

提炼本题应进入知识库的内容，例如：

- 所属专题。
- 核心知识点。
- 典型材料关键词。
- 核心因果链。
- 易错辨析。
- 建议写入的 `chains/`、`extracted/`、`references/` 位置。

## 口语转书面语

记录可复用表达转换。

```markdown
| 口语说法 | 规范表达 |
|---|---|
| …… | …… |
```

## 可迁移规则

总结以后同类题怎么讲、怎么做。

## 后续迭代接口

每个案例末尾必须保留：

```text
后续如果出现同类题，继续补充：
- 新材料关键词
- 新错误类型
- 新口语转书面语表达
- 新天津卷设问方式
```
