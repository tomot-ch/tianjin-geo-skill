# 聊天记录双线蒸馏 Prompt

你要处理用户提供的地理讲题聊天记录。

必须分两条线输出：

## 线1：讲题风格 / 口语化

提炼：

- 老师如何开头。
- 如何让学生回到材料。
- 如何纠错。
- 如何讲因果链。
- 如何把专业词讲成人话。
- 如何口语转书面语。
- 如何总结可迁移方法。

输出到：

- `references/my_oral_style_summary.md`
- `references/my_oral_phrase_bank.md`
- `templates/oral_teaching_template.md`
- `templates/oral_to_written_template.md`
- `examples/my_teaching_cases/`

## 线2：知识点 / 题型积累

提炼：

- 知识点名称。
- 所属专题。
- 典型材料关键词。
- 核心因果链。
- 易错辨析。
- 天津卷常见问法。
- 可迁移题型。
- 应写入的知识库位置。

输出到：

- `references/case_knowledge_accumulation.md`
- 对应 `chains/*.md`
- 对应 `extracted/*.md`
- `references/knowledge_framework.md`
- 必要时更新 `templates/tianjin_language.md`

## 严格要求

1. 两条线必须分开。
2. 不要把聊天原文直接塞进知识库。
3. 不要保存学生隐私。
4. 不确定的知识点要标记为“待核查”。
5. 对用户纠正过的信息必须覆盖旧理解，例如：韭园是陕西，不是尼泊尔。
6. 不要删除原有内容，只能追加或小范围修正。
7. 每次更新都要在文件末尾保留“后续迭代接口”。

## 四、建议写入仓库位置

## 五、给 Codex 的增量更新指令
